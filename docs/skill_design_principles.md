# Skill Design Principles (SOLID)

Agentic OS (`agent-core`) において、AIエージェントの振る舞い（Skill）を実装する際の絶対的な設計原則を定義する。

## 1. 2-Tier Architecture (オーケストレーターとワーカーの分離)

Skillは、責務に応じて「オーケストレーター層」と「ワーカー層」の2段階に明確に分割すること。

*   **Tier 1: Orchestrator (状態管理と委譲)**
    *   **責務**: 処理の全体の流れ（Workflow）を定義する。
    *   **特徴**: ロジックを持たない。ユーザーとの対話が必要な場合や特定の処理が必要な場合は、自ら実行するのではなく、Tier 2 のスキルを持つ「サブエージェント（Subagent）」を起動（`invoke_subagent`）し、タスクを委譲する。
*   **Tier 2: Worker (個別タスクの実行)**
    *   **責務**: 与えられた単一の専門的なタスクを完璧にこなす。
    *   **特徴**: 他のプロセスを知らない。入力をもらい、成果物を出力する（またはユーザーと対話する）ことに特化する。

## 2. SOLID原則の適用

1.  **Single Responsibility Principle (単一責任の原則)**
    *   1つのSKILL（特にTier 2）は、1つのタスクだけを行う。
    *   例: 「深掘り対話をしてフォーマットも整える」スキルはNG。「深掘り対話（Interviewer）」と「フォーマット（QA）」に分割する。
2.  **Open-Closed Principle (開放閉鎖の原則)**
    *   既存のスキルを修正せずとも、新しいサブエージェントをオーケストレーターのフローに追加するだけで機能を拡張できるようにする。

## 3. Zettelkastenとの境界線の厳守

*   **Skillは振る舞い、Zettelkastenは知識**。
*   Skillのプロンプト内に「会社の方針」や「特定のプロジェクトの前提知識」をハードコードしてはならない。それらは `10_Areas` や `40_Permanent_Notes` から JIT (Just-In-Time) でロードさせること。
