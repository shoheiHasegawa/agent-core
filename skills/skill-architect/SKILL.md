# Skill: Skill Architect

## 🎯 目的
Agentic OSのエージェントが使用する「Skill」を設計・生成する。システムプロンプトの記述からディレクトリ構成の作成までを一貫して行う。

## ⚠️ 絶対遵守ルール
スキルを設計・実装する前に、**必ず** `agent-core/docs/skill_design_principles.md` （Skill Design Principles）を読み込み（`view_file`）、以下の原則に違反していないかをチェックすること。
1. **2-Tier Architecture**: 作成しようとしているスキルは「オーケストレーター（Tier 1）」か「ワーカー（Tier 2）」かを明確に定義すること。
2. **SOLID (単一責任)**: ワーカーの場合、複数の作業（例：リサーチしてドキュメントを書く）を1つのプロンプトに混ぜ込まないこと。
3. **Zettelkastenの独立性**: スキル内に固有のドメイン知識をハードコードしないこと。
4. **立法と司法の分離 (Separation of Concerns)**: スキル内に特定のドメインルール（DDDの制約やハードコード禁止等）を直接記述してはいけない。ルールは `docs/rules/` 等のドキュメントに外部化（立法）し、スキルは「それを動的にロードして評価する」アルゴリズム（司法）のみを実装すること。

## 🛠️ 実行手順
1. **要求定義**: ユーザーがどのようなスキルを必要としているか（目的、インプット、アウトプット）を分析する。
2. **設計案の提示**:
   - スキル名と概要
   - Tier（Orchestrator か Worker か）
   - ディレクトリ構造と必要なリソースファイル（`scripts/` 等が必要か）
   をユーザーに提案する。
3. **実装**:
   - ユーザーの承認を得た後、`agent-core/skills/<skill_name>/SKILL.md` を作成する。
   - YAMLフロントマター（`name`, `description`）を必ず記述する。
4. **レビュー依頼**:
   - 作成後、ユーザーに対して「`skill-reviewer` スキルを使って品質チェックを行ってください」と提案する。
