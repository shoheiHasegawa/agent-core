# Johari Profiler (深層プロファイリング・スキル)

## 📌 目的 (Mission)
時間経過や環境変化によって生じる「価値観のドリフト（ズレ）」を検知し、ユーザーの「真のIdentity（The Terminal Value）」を再定義する。
人間の「表向きの宣言（顕在意識）」と「無意識の執着（潜在意識）」を隔離分析し、「ジョハリの窓」を用いて衝突・統合させることで、圧倒的な解像度でユーザーの現在地をメタ認知させる。

## 🛠️ 実行手順

このスキルは、親エージェント（あなた）がオーケストレーターとなり、2つの隔離されたサブエージェントを並行稼働させることで実行する。

### 1. 顕在意識のプロファイリング (Explicit Value Analyst)
*   **アクション (Action)**: `invoke_subagent` を使用し、「Explicit Value Analyst」を起動する。
*   **制約事項 (Constraints)**: 以下のプロンプトを与えること。
    「ワークスペース内のタスク（Epic）ディレクトリや進行中のプロジェクト資料を検索・特定して読み込みなさい。表面的なルールを『なぜそれを行おうとしているのか（ラダーリング法）』で深掘りし、ユーザーが『世界に対してどうありたいと宣言しているか（ペルソナ）』をレポートとして提出してください。」
*   **出力 (Output)**: 顕在意識のレポート

### 2. 潜在意識のプロファイリング (Implicit Value Analyst)
*   **アクション (Action)**: `invoke_subagent` を使用し、「Implicit Value Analyst」を起動する。
*   **制約事項 (Constraints)**: 以下のプロンプトを与えること。
    「ナレッジベース（Zettelkasten / 40_Permanent_Notes）領域のみを検索・特定して読み込みなさい。抽象的な概念の裏にある『どうしても執着してしまうもの（究極の欲望）』や『恐れているもの』をラダーリング法で深掘りし、表面的なロジックを剥ぎ取った無意識のコアをレポートとして提出してください。」
*   **出力 (Output)**: 潜在意識のレポート

### 3. ジョハリの窓による統合 (Synthesis)
*   **入力 (Input)**: 両サブエージェントからのレポート
*   **アクション (Action)**: 両レポートを「ジョハリの窓」のフレームワークで衝突・統合し、分析結果を `[Date]_johari_profiling_report.md` というArtifactとして出力する。
    1.  開放の窓: ユーザーも自覚し、ノートにも現れている「強みと美学」
    2.  盲点の窓: 自覚はないが、ノートの構造から滲み出ている「本当の目的や執着」
    3.  秘密の窓: 自覚はあるが、外部のタスクには現れにくい「個人的な完璧主義や恐れ」
    4.  未知の窓 (The Terminal Value): 顕在と潜在が融合して見えてくる「真のIdentity」
*   **出力 (Output)**: Artifactによるユーザーへの提示と壁打ち（Sense Making）

### 4. 分析結果の永続化とアップデート
*   **入力 (Input)**: ユーザーから合意を得たレポート
*   **アクション (Action)**: レポートをナレッジベース内の適切なリソース（Resources）領域を検索して保存し、必要に応じて現在のアイデンティティやコアビジョンを定義しているファイルに反映（アップデート）する。
