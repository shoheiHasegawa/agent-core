# 概要
`Mail_Content_Converter` プロジェクト（旧：Phase 3タスク）から分離したインフラ・セキュリティ改善案件。

## 課題
個人の自動化システム（Pythonスクリプト等）でIMAPを用いてメールを取得する際、メインのGoogleアカウントで発行した「アプリパスワード」を直接使用している。
万が一 `.env` 等の認証情報が流出した場合、メインアカウント内の全メール（個人情報・決済情報など）が閲覧可能となり、パスワードリセット機能を通じた別サービスの乗っ取り等の甚大な被害に繋がる。

## 次のアクション
「被害の極小化」を前提としたアーキテクチャへ移行する。
1. **専用BOTアカウントの作成**: スクリプトがアクセスするための専用アカウント（例：`shohei.bot.finance@gmail.com`等）を作成。
2. **自動転送の活用**: メインアカウント側で設定しているフィルタを修正し、処理対象となるメルマガ等の特定メールのみをBOTアカウントへ自動転送する。
3. **認証情報の差し替え**: プログラム側（自動化スクリプト）の接続先と `.env` をBOTアカウントのものに書き換え、稼働を確認する。
4. **旧設定の破棄**: 移行完了後、メインアカウント側のアプリパスワードを速やかに無効化・削除する。
