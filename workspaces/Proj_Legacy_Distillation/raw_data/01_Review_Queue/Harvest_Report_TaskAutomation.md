# Harvest Report: Task & Calendar Automation 上流設計セッション

**Date**: 2026-05-30
**Topic**: モバイルUIを起点としたタスクと予定の自動化アーキテクチャ設計

## 🌾 Wisdom (得られた教訓・普遍的な知見)
*   **「時間」と「文脈」の分離が摩擦をなくす**: カレンダーは「何時に何をやるか（時間枠）」に特化させ、具体的な作業リストやコンテキストはObsidianに切り出す。両者をObsidian URI（ディープリンク）で接続することで、強力なモバイルUIが実現できる。
*   **カレンダーは完了（Done）を管理する場所ではない**: 予定は時間がくれば消滅するため、タスクの進捗管理とは相性が悪い。完了管理は「ジャーナリング」や「Obsidianのタスクリスト」に完全に委譲することで、UIの役割がクリーンになる。
*   **社会構成主義とジャーナリングのシステム化**: 単なるタスク消化ではなく、「なぜ予定通りにいかなかったか」「どう感じたか」を自分の言葉で言語化する（Coach Agentとの対話）。この「自分の言葉が世界観を作る」という内省のプロセスこそが、本質的な生産性（Well-being）の向上に繋がる。
*   **実行環境の制約を逆手に取る**: 「昼間はMacがオフ」という制約があるからこそ、リアルタイム同期を捨てて「夜間バッチ（launchd）」による計画・内省フェーズを独立させることができ、心理的なオンオフの切り替えにも寄与する。
*   **Agentによる休息の強制と自律的成長**: ストイックな性質のユーザーに対し、システムは単にタスクを詰め込むのではなく「休息のスケジューリング」を担う必要がある。予定と実績の乖離（タスクが大きすぎた等）をAgentが分析し、自身のプロファイル（`Accumulated Wisdom`）を毎晩自己更新することで、日を追うごとに「無理のないタスク粒度の分解」ができるように自律成長させるアーキテクチャが極めて有効である。

## 🛠️ Tech Debt (技術的負債・今後の懸念事項)
*   **iOSリマインダー連携の壁**: リマインダーのUIは強力だが、Mac CLIからリマインダーへタスクを双方向同期させるのはAppleのエコシステム上困難（AppleScript等が必要で壊れやすい）。初期フェーズではスコープ外としたが、完了時の「チェックを入れる快感」の代替UIについて引き続き検証が必要。
*   **割り込みタスクの手動記録の手間**: 割り込みタスクの実績データは重要だが、ユーザーが直接iOSカレンダーに書き込む運用とした。これが長期的に入力負担とならないか、運用の様子を見る必要がある。
