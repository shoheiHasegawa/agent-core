## 🧠 Accumulated Wisdom (得られた教訓と決定事項)

### The Ultimate Inbox Pipeline (双方向の非同期パイプライン)
「Inbox」は単なるゴミ箱ではなく、人間とAgentが非同期にタスクを依頼し合い、アイデアを熟成させるための「パイプライン（流れる川）」である。
このパイプラインを視覚的かつ強制的に機能させるため、`second-brain/00_Inbox/` 配下のディレクトリに順序を示すプレフィックス（00〜03）を付与し、以下の責務（窓口）を定義した。

1. **`00_Drop_Zone/` (共通の受付窓口)**
   - 人間やモバイルからの未整理な入力（カオス）が着弾する場所。Agentが毎朝ここを読み取り、以下の窓口へディスパッチする。
2. **`01_Agent_Queue/` (人間 👉 Agent への依頼窓口) [NEW]**
   - 「〇〇を調査して」「コードを書いて」など、AIが自律実行できる非同期タスクの待ち行列。Worker Agentがここを監視して実行する。
3. **`02_Review_Queue/` (Agent 👉 人間 への報告窓口)**
   - AIが実行したタスクの結果や自律生成したHarvest Reportなど、人間による確認と承認を待つ検疫エリア。
4. **`03_Idea_Backlog/` (アイデアの育成窓口) [NEW]**
   - 今すぐ行動しないが価値のある知見が沈殿する場所。後にZettelkasten（ナレッジ）として昇華するための種置き場。
5. **`Inbox_Tasks.md` (人間がやるタスク)**
   - 物理世界での行動（市役所に行く等）を要する、人間のための実行待ちリスト。COOモジュールがカレンダーに配置する。

### Zettelkasten的「設計思想」の昇華プロセス
アーキテクチャの閃きや重要なルールは、この「Harvest Report」として `02_Review_Queue` に種として生み出され、人間の承認を得たのち、以下の2箇所に分裂して永続化される。
- **How (Agentのルール)** 👉 `.agent/` 配下の憲法（AGENTS.mdなど）へ追記される。
- **What/Why (人間のマニュアル)** 👉 `02_Areas/` 等の恒久的なシステム設計書として保管される。
