## 📌 1. 効率的・再現性のある学習モデル（棚卸し）

### 💡 勉強方針の全体像
* **能動的に思い出す（想起学習／アクティブリコール）**：ただ読み返す、下線を引く、書き写す作業は「脳への負荷が少なすぎる」ためNG。
* 声に出しながら参考書を読み（20分）➡️ 何も見ずに書き出し（10分）➡️ 補完する。
* **精緻化質問**：自分に「なぜ？」「どうやって？」と問いかけることで理解を深める。

### 📊 試験ごとの対策方針（時間配分イメージ）
* **午前Ⅰ（共通）**：応用情報の過去問（80問中30問出題）を過去問道場で反復。
* **午前Ⅱ（専門）**：過去問（約235問等）を 1問3分 でテンポよく周回。 GoodNotes等で間違えた問題をフラッシュカード化。
* **午後（記述・論文）**：午後1（必須問題）➡️ 午後2（記述・論文問題）の順で優先度を上げる。
  * アルファベット用語は DNS ＝ Domain Name Service のように正式名称から仕組みを深く覚える。

## 📌 2. 資格維持と費用（セキスペの収支）

情報処理安全確保支援士（SC）は3年ごとの更新義務があり、講習費用との収支バランスを把握しておくこと。

## 📌 3. プロジェクトマネージャ（PM午後2）論文ネタ・引き出し

試験時のエッセンス（独自性とテーラリング）を組み立てるための要素。

### 🛠️ テーラリング（Pマネジメント標準からの調整）
* テンプレートはあくまで型であり、全く同じプロジェクトは存在しない。
* 自分の動きを「上位の視座」に置き換えて書くこと（やったこと ➡️ 指示したこと）。

### 👥 混成チーム・チーム管理でのプロセス提案
* **コミュニケーション**: 
  * 派遣・パートナー混成チームの場合、個別面談（動機づけ）、全体会議への全員参加、窓口担当者の設定などをプロセス化する。
  * 法律と契約に関する説明会によりルールのすり合わせ。
* **ジュニアエンジニア支援**: 設計思想や業務知識の不足を補うため、AI活用等のアプローチを推進して底上げを図る。

### 🚨 進捗管理とトラブルシューティング
* **納期遵守**: 課題の吸い上げ漏れが最大のリスク。工程をサブタスクに分解して開始・終了日程のモニタリング。
* **不具合対策**: 要件定義での解釈ズレ（暗黙用語、旧プログラム仕様のズレ等）による手戻りが多い。本番データによる事前の成果物すり合わせ等で防ぐ対策をもつ。

## 📌 4. 技術レポート・論文の書き方（PREP法）

### 📝 レポート・論文記述のルール
* **感想文にしない**: 「〜だと思う」はNG。**「〜である」「〜だ」** と言い切る。
* **PREP法**: Point（結論）➡️ Reason（理由）➡️ Example（事例）➡️ Point（結論）。結論から書く。
* **活用と成果の対応**: 成果というより「今後の活用と反省」に逃げていないかチェック。実践した成果をアピールする。

### 🔍 QCDと過剰品質への意識
エンジニアとして「過剰品質もNG」であることを念頭に置く。目標を最低限満たすレベル（100点と4点でコストが同じなら、合格圏内を最小コストで通す）のさじ加減を意識する。
