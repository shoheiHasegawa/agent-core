# アイデア：トレード日誌の「撮り溜め一括処理」モード

## 背景
現在、`trading-automation` は iPhone の a-Shell 上で、撮影直後の 3 枚をその場で処理する運用となっている。将来的に Mac 側（`life-automation`）で大量の過去ログを整理したり、忙しくてその場で処理できなかったスクショを後から一括変換したくなる可能性がある。

## 提案ロジック
1.  **スキャン**: Inbox ディレクトリ内の画像ファイルを撮影日時順にソートして取得。
2.  **グルーピング**: 3 枚 1 セット（週足・日足・30分足）として画像をグルーピング。
    - 撮影日時が近い 3 枚をセットとみなす、あるいは単純に 3 枚ずつ取り出す。
3.  **連続生成**: ループを回して、画像セットごとに Markdown 日誌を自動生成。
4.  **整理**: 処理が完了した画像は自動的に `Attachments` へ移動。

## 期待される効果
- **摩擦のさらなる低減**: その場でスクリプトを叩く時間がない時でも、後から `make cfo-trade-log` 一発で日誌が完成する。
- **分析基盤への統合**: 過去のトレードを網羅的にログ化することで、AI によるトレード傾向分析の精度が向上する。
