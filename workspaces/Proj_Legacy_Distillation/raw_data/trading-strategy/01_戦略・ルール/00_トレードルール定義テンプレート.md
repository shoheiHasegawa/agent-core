# 📋 トレードルール定義（プロンプト型テンプレート）

このノートは、トレードルールを「システム（AIまたは自身の脳）がブレなく実行できるレベル」で言語化するためのフォーマットです。

> [!IMPORTANT]
> すべての条件は **「True / False で判定可能な表現（定量的な数値や明確なファクト）」** になるよう定義してください。曖昧な形容詞（強い、押し目、少し等）は排除します。

---

## 🎭 1. 役割（Role）
あなたは **[ 例: 規律を厳守するスイングトレーダー / 冷徹な売買執行ボット ]** です。
*   **目的**: [ 例: 月利〇%の達成 / 許容リスク内での利益最大化 ]
*   **基本方針**: ルールにない行動はすべてバグ（エラー）として処理する。

---

## 🌍 2. 【Context】環境認識（前提条件）
*戦っても良いフェーズ（相場環境）か？*

| 判定項目 | 判定条件（True/False） | 確認ツール・指標 |
| :--- | :--- | :--- |
| **全体地合い** | 例) 日経平均が 200日SMA より上にあるか | 日足チャート |
| **ボラティリティ** | 例) 日経平均VIが `20` 未満であるか | VI指数 |
| **セクター動向** | 例) 該当セクター指数が前日比プラスであるか | セクターチャート |

---

## 🎯 3. 【Setup】セットアップ（監視候補一覧）
*候補（ユニバース）を選定するための、罠を張るべき状態。*

*   [ ] **流動性**: 過去1ヶ月の平均売買代金が `〇〇億円` 以上である。
*   [ ] **環境**: 日足の長期RCI(81)が `0` 以上（上向き）である。
*   [ ] **サイン1**: 日足の短期RCI(9)が `-80` 以下のエリアに到達した。
*   [ ] **サイン2**: ボリンジャーバンド(25)の幅が収縮（スクイーズ）している。

---

## 🔫 4. 【Trigger】執行条件（エントリー引き金）
*どの瞬間にボタンを押すか？（解釈の余地をゼロにする）*

*   **トリガー条件 (Entry Signal)**:
    *   [ ] 例) 日足の短期RCI(9)が上向きに転じた（前日値 < 当日値）
*   **発注アクション (Execution)**:
    *   [ ] 例) 翌営業日、**[ 前日高値 + 1円 ]** を超えたら成行買いの「逆指値（IFD）」をセットする。

---

## 🚪 5. 【Exit & Risk】出口とリスク管理
*いくらで戦い、どこで決済・撤退するか？*

*   **リスクコントロール**:
    *   [ ] **ロット（株数）計算**: `(総資金 × 1%) ÷ (エントリー想定価格 - 損切り価格)`
*   **損切り (Stop Loss)**:
    *   [ ] 例) エントリー根拠となったサポートライン（直近安値等）の `-2円`
*   **利益確定 (Take Profit)**:
    *   [ ] 例) ボリンジャーバンドの `+2σ` タッチ、または RCI(9) が `+80` での折れ曲がり時

---

## 🚫 6. 【Constraints】禁止・除外事項（Negative Prompt）
*これに1つでも該当すれば、エントリーをキャンセル（Return False）するシステムバグ。*

*   [ ] **イベント**: 決算発表まで `3営業日` 以内である。
*   [ ] **時間帯**: 寄り付き直後（9:00〜9:15）の発注（ノイズ回避）。
*   [ ] **指標不適合**: 長期移動平均線が下を向いている。

---

## ✅ 運用チェックリスト
*(毎回のトレード前にチェックしてロボットのように執行する)*
- [ ] Context を満たしているか？
- [ ] Setup が完了しているか？
- [ ] Constraints に1つも該当しないか？
- [ ] ロット計算がルール通りか？
