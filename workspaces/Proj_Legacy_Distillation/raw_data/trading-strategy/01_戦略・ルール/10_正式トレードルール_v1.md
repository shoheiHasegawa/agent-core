# 📋 トレードルール定義：v1.0 (Rule of 3 / 4 Pillars)

このノートは、4つの主要指標（パラボリック、ボリンジャーバンド、RCI、出来高）を軸とした、執行精度の高いトレードルールを定義したものである。

---

## 🎭 1. 役割（Role）
あなたは **「感情を排し、統計的優位性のみを執行する規律型スイングトレーダー」** です。
*   **目的**: 3の倍数フラクタルに基づく大局の波に乗り、許容リスク内（総資金の1%）で利益を最大化する。
*   **基本方針**: 楽天証券のツール制約を理解しつつも、裏側のロジック（Pythonスクリーナー）による厳密なフィルタリングを最優先する。

---

## 🌍 2. 【Context】環境認識（戦場の判定）
*以下の条件をすべて満たす場合のみ、戦場として認める（True）。*

| 判定項目 | 判定条件（True/False） | 確認ツール |
| :--- | :--- | :--- |
| **ボリンジャーバンド(27)** | 中心線（基準線）が **「明確に上向き」** であること | 日足 / 楽天証券 |
| **長期RCI(81)** | 指数が **「0以上」** または **「上向き」** であること | Pythonスクリーナー |
| **中期RCI(27)** | 環境確認として **「上向き」** であること | Pythonスクリーナー / 楽天証券(※) |

(※) 楽天証券で表示可能なRCIが2本の場合、短期(9)と長期(81)を表示させ、システム的な中期判定を優先する。

---

## 🎯 3. 【Setup】セットアップ（監視候補の抽出）
*以下の条件が重なり、罠が張られた状態を待つ。*

*   [ ] **流動性**: 東証プライム市場の時価総額上位、または売買代金が十分な銘柄である。
*   [ ] **ノイズ除去**: **「3日合成足」** において、直近のトレンドに逆行する不自然な動き（バグ）がない。
*   [ ] **冷却**: 短期RCI(9)が **`-80` 以下** のエリアに到達し、過熱感が完全に冷却されている。
*   [ ] **ボラティリティ**: BB(27)がスクイーズ（収縮）から **エクスパンション（拡大）** の初動、または $\pm 2\sigma$ 内での健全な推移である。

---

## 槍 4. 【Trigger】執行条件（エントリー引き金）
*解釈の余地なく、以下の瞬間にボタンを押す。*

*   **トリガー条件 (Entry Signal)**:
    *   [ ] 日足の短期RCI(9)が **下から上に転じた（前日値 < 当日値）** ことを確認。
    *   [ ] 当日の **「出来高」** が前日比、または3日平均に対して **急増** している（大口の介入）。
*   **発注アクション (Execution)**:
    *   [ ] 翌営業日の寄付き前、**[ 前日高値 + 1円 ]** を超えたら成行で買う **「逆指値（IFD）」** をセットする。損切りラインも同時に予約する。

---

## 🚪 5. 【Exit & Risk】出口とリスク管理
*いくらで戦い、どこで降りるか。*

*   **リスクコントロール**:
    *   [ ] **1トレードの最大損失**: 総資産の **`1%`** を上限とする。
    *   [ ] **ロット計算**: `(総資産 × 1%) ÷ (エントリー想定価格 - 損切り価格)`
*   **損切り (Stop Loss)**:
    *   [ ] **パラボリック(SAR)** の点。またはエントリー根拠となった直近安値下。
*   **利益確定 (Take Profit)**:
    *   [ ] **パラボリックの反転（ヒット）** を決済の絶対基準（トレイリングストップ）とする。
    *   [ ] 第1ターゲットはボリンジャーバンドの **`+2σ`** タッチ、または真空地帯（次のシグマ線）の終端。

---

## 🚫 6. 【Constraints】禁止事項
*以下の1つでも該当すれば、システムエラーとして参戦を拒否する。*

*   [ ] **BBスクイーズ中の迷走**: バンドが収縮し、方向感がない中での空中戦。
*   [ ] **魔の時間帯**: 9:00〜9:15 の板が薄い時間帯での飛び乗り（翌朝逆指値で回避）。
*   [ ] **出来高なき上昇**: 株価が上がっても出来高が細っている（燃料切れ）。
*   [ ] **SAR逆行**: ローソク足がパラボリックの下にある状態での買い。
*   [ ] **日経平均への逆張り（マクロ非同期）**: 相場全体が下落している日に、個別株のセットアップだけで買うこと。
*   [ ] **感覚的な利益の先延ばし**: 目標到達時の「もっと上がるかも」という欲による放置。必ず逆指値を引き上げ（トレール）、勝ち確状態を作る。

---

## 🔄 7. 【Protocol】全工程マスターフロー
*実際のトレード業務全体を管理するマニュアル*

#### Phase 0：【前提確認】（戦う前の準備）
- **資金枠の確認**：今回のトレードに使える余力は？
- **許容リスクの決定**：総資産の1%以内（最大損失額）の決定
- **環境確認**：ザラ場を見れるか？（見れないならリンク注文必須）

#### Phase 1：【銘柄選定】（スクリーニング）
- **待ち伏せ**：Watchlistの中で、RCIが底打ちした銘柄を定点観測
- **形状変化**：BBがスクイーズ（収縮）してエネルギーを溜めている銘柄の抽出
- **セクター**：資金の流れ（業種テーマ）の確認

#### Phase 2：【シナリオ構築】（※最重要プロセス）
1. **狙いの決定**：月足レベルの「ホームラン」か、レンジ内の「ヒット」か
2. **損切りラインの特定**：チャート上の節目（割れたら終わりのライン）を特定
3. **ポジションサイジング**：`許容リスク ÷ (エントリー想定値 - 損切り値) ＝ 株数` の計算・調整

#### Phase 3：【エントリー執行】（注文の技術）
- **打診買い**：いきなり全力ではなく、半分（試し玉）から入る
- **時間分散**：魔の時間（9:00〜9:15 / 12:30〜12:40）の執行回避
- **注文種別**：IFD等の「逆指値（ここまで上がったら買う）」の活用

#### Phase 4：【ポジション管理】（オーダーシフト戦略）
- **ガードの徹底**：買った直後に「リンク注文（OCO）」で利食いと損切りを同時セット
- **魔の時間帯のオーダーシフト（逆指値のずらし）**：
  1. **9:00前 / 12:30前**: 逆指値を「深い位置（災害ストップ用）」に一時的に下げる（12:30のアルゴ狩りの回避）。
  2. **波乱通過後（9:30以降 / 12:40以降）**: 逆指値を「本来のテクニカルな損切り位置」に戻す。
- **増し玉（ピラミッディング）**：トレンドが確定したら残りの資金を投入

#### Phase 5：【決済戦略（Exit）】
- **分割利食い**：最初のレジスタンスで半分利食いし「負けなし」を作る
- **トレーリングストップ**：残りはパラボリックを基準に逆指値を切り上げ限界まで伸ばす
- **ルールの遵守**：損切りラインに触れたら感情を殺して即座に切る

#### Phase 6：【振り返り】（成長のサイクル）
- **答え合わせ**：「エントリー根拠は正しかったか？」「資金管理は適正だったか？」「ノイズに狩られなかったか？」
- 次回への改善点を日誌にメモする
