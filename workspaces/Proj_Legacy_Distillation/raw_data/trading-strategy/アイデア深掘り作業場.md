# 🛠️ トレードルール構築：アイデア深掘り作業場

このノートは、過去の Inbox (`01_Inbox/CFO/01_戦略・ルール`) に蓄積されたアイデアをプロンプトフレームワークに分類し、ルールとしてブラッシュアップするための作業場です。

---

## 👑 確定したコア・ロジックとシステム設定値

相場の「静と動」「エネルギーとタイミング」を捉えるため、2つの相互補完グループをベースシステムとします。

### 1. 指標の組み合わせ（コア・ロジックの思想）
*   **グループA: 「ボリンジャーバンド ＋ パラボリック」 [フィルター ＆ トリガー]**
    *   **役割**: 相場のボラティリティ（変動率）とトレンド方向、決済ラインの管理。
    *   **思想**: パラボリック最大の弱点（レンジでのダマシ）を、BBのスクイーズ（収縮）/エクスパンション（拡大）で完全にフィルターする。**BB $\pm 2\sigma$ ブレイク時のサインのみを採用**。
*   **グループB: 「出来高 ＋ RCI」 [エネルギー ＆ タイミング]**
    *   **役割**: 相場の大衆の熱量（大口の痕跡）と波のサイクルを測定。
    *   **思想**: RCIで波のサイクル（タイミング）を測り、**出来高の急増（大口の介入）でシグナルの「裏付け」をスコアリング**する。

### 2. システム設定値（システム仕様）
最小単位を「3の倍数（ノイズ希釈）」で統一するフラクタル設計をベースとします。

| 指標 | 期間 / 設定値 | 役割 |
| :--- | :--- | :--- |
| **ボリンジャーバンド (BB)** | 期間 **`27`** / **$\pm 2\sigma$** | 相場環境の判定、ダマシのフィルタリング |
| **RCI** | **`9`** (短期) / **`81`** (長期) | エントリータイミング、ダイバージェンス検知 |
| **パラボリック (SAR)** | 加速因数 **`0.02`** / 最大値 **`0.20`** | 自動決済ラインの追従（トレイリングストップ） |

### 3. ボリンジャーバンド（BB）の運用ルール
*BBの形状から環境を「True / False」で判定する基準*

*   **環境定義（Context）**:
    *   **横向きのBB**: レンジ相場
    *   **傾いているBB**: トレンド相場（傾きの方向 ＝ トレンドの方向）
*   **フィルタリング基準（Setup）**:
    *   **スクイーズ（収縮）**: 値幅が取れないため **「戦いの場ではない」として参戦禁止（除外）**。
    *   **線の向き（抵抗感）**: 横向きの線は越えやすく、傾いた線は強いサポート（下抜けに努力がいる）として機能する。
*   **禁止事項（Constraints） ⚠️**:
    *   **横向き（レンジ）の基準線付近からのエントリー禁止**: 方向感が完全に消失する「台風の目」であるため。
*   **基本戦略・ターゲット設定（Exit / Trigger） 🎯**:
    *   **トレンド**: バンドウォーク（$\pm 2\sigma$等のバンドに沿った推移）を維持。
    *   **真空地帯（抵抗のないエリア）**: バンドウォーク以外の状況では、**「次のBBの線（シグマ線・基準線）」までを真空地帯** と認識する。例) $+1\sigma$ を下回ったら、ターゲットは基準線まで（幅が狙える値幅になる）。
    *   **反転引力**: BBがエクスパンションしている状態で **「基準線に戻ろうとする引力」** をエントリー・エグジットのシナリオに利用する。

### 4. RCI（短期・長期）の運用ルール
*RCIを「タイミング」と「リズム」の確認、および「風向き」のデバッグとして使う基準*

*   **前提認識（遅行性の理解） ⚠️**:
    *   短期・長期ともに **「遅行性」** であることを大前提とする（値動きについてくるだけの答え合わせ）。
    *   **大引けで確定する**: 特に短期RCIは **クローズするまで当日中は当てにならない**。ザラ場中の単体シグナルでの飛び乗りは禁止する。
*   **短期 RCI(9) の役割**:
    *   **タイミングとリズムの可視化**: 上向＝上昇継続、折れ曲がり＝タイミング/リズムの変化（「道を間違えた、風向きが変わった」ことの自己デバッグに使う）。
*   **長期 RCI(81) の役割**:
    *   **期待値のブースト**: **「短期RCIが、長期RCIの方向についてくる（シンクロする）」** 状態になると、トレードの期待値（信頼性）が段違いに跳ね上がる。

### 5. パラボリック（SAR）の運用ルール
*パラボリックを「トレンドの息切れ（手仕舞い）」の絶対防衛線として使う基準*

*   **本質的理解（追いかけっこ） 🏃‍♂️**:
    *   パラボリックは **「ローソク足（価格）を追いかける指標」** である。価格が更新され続ければ逃げ切れるが、追いつかれた（交差した）時点で **「トレンドの勢いがなくなった（息切れ）」** と判定する。
*   **決済（Exit）での活用**:
    *   徐々に加速するパラポリックに対して、ローソク足の逃げ足はいつか鈍化（息切れ）する。この **追いつかれた瞬間を決済のタイミング** とする。
*   **禁止事項（Constraints） ⚠️**:
    *   **パラボリックの反転を「新規エントリー」のトリガーにしてはならない**。パラボリックは「トレンドの終わり」を示すものであり、次の展開（休憩か転換か）は保証しない。
    *   **トレンドの発生条件**: あくまで **「価格が平行線を抜けた（高値更新、または安値更新）」** 事実のみとする。

## 🎭 1. 役割 (Role) / マインドセット
*心構え・規律に関するアイデア*

*   **規律の実行力（ロボット化）**:
    *   「分かっているけどできない」をゼロにする。恐怖や欲を排除し、決めたルールをロボットのように執行する。
    *   ルールにない行動（ポジポジ病、空中での飛び乗り）はすべて **「システムバグ（エラー）」** として検知・ブロックする。

## 🌍 2. 【Context】環境認識
*戦っても良いフェーズ（大局・波）を決めるためのアイデア*

*   **3の倍数パラメータによる波の同調**:
    *   中期RCI(27)の方向感、長期RCI(81)の大局観を重視。
    *   「3日」をノイズが希釈される最小単位とし、合成足でノイズを除去するアプローチ。
*   **BB(27) の幅感**:
    *   ボリンジャーバンドの幅（スクイーズ/エクスパンション）でエネルギーの充填・放出を測る。
    *   **統計的確率の意識**: 価格が $\pm 2\sigma$ 内に収まる確率は約95.4%。**トレンド発生（バンドウォーク）の検知** や、過熱感のフィルターとして活用する。
*   **抵抗帯（ボリュームゾーン）の特定**:
    *   出来高が数日間より多い価格帯は反転（レジサポ）の強力な根拠になる。

## 🎯 3. 【Setup】セットアップ
*監視候補を選定し、罠を張るためのアイデア*

*   **自動スクリーニング（ポジポジ病の防壁）**:
    *   銘柄への執着を捨て、全銘柄（フィルタリング後）を横断してセットアップ完了銘柄を機械的に抽出する。
*   **RCIの引き付け**:
    *   短期RCI(9)が `-80` 以下にしっかりと突っ込んでいること（過熱感の冷却）。
*   **BBの中心線維持**:
    *   BB(27) の基準線の「上」で株価が定着し、割らないことを確認する。
*   **出来高による「裏付け」と信頼性**:
    *   レジスタンス突破時、出来高が急増していれば「本物のブレイク（大口参戦）」。伴っていなければセットアップ保留。

---

## 🔫 4. 【Trigger】執行条件
*ボタンを押す、または注文を予約するトリガーのアイデア*

*   **RCIの切り上がり**:
    *   短期RCI(9)が `-80` 以下から「上向きに転じた（前日値 < 当日値）」瞬間。
*   **大引けノイズの回避（翌朝執行）**:
    *   日足大引けの出来高はインデックス等の機械的注文で歪むため、大引けでの即時エントリーを禁止。
    *   **逆指値（ストップ買い）の自動化**:
        *   「前日の高値を 1円 でも上抜けたら成行買い」の逆指値（IFD）をセットし、本物の買いエネルギーが来た場合のみ自動で乗る仕様。

---

## 🚫 5. 【Constraints】禁止・除外事項
*絶対にやってはいけないこと（Negative Prompt）*

*   **魔の時間帯（タイムセグメンテーション対策） ⚠️最重要**:
    *   **`9:00〜9:15` および `12:30〜12:40` の注文執行を避ける。** 板が薄く、アルゴリズムの逆指値狩りに合いやすい。
    *   **対策**: 11:30（前引け）に逆指値を一旦外す、または深くする運用。
*   **騙しの急騰**:
    *   売りの勢力を吸収し切る（レンジ等の仕込み）がないのに急騰しているチャートは、高値掴み（大口の売り抜け拠点）の可能性が高いためエントリー禁止。
*   **出来高のダイバージェンス（燃料切れ）**:
    *   株価が高値を更新しているのに出来高が減少している場合、上昇エネルギーの枯渇としてエントリーを見送る。

## 🚪 6. 【Exit & Risk】出口とリスク管理
*いくらで戦い、どこで決済するか*

*   **ポジションサイジング（数量の計算）**:
    *   `許容リスク幅 ÷ (エントリー価格 - 損切り価格) ＝ 株数` を厳守。
*   **分割売買（スケーリング）**:
    *   いきなり全力買いせず、まずは **「打診買い（半分）」** から入る（リスク分散）。
    *   トレンドが出たら **「増し玉（ピラミッディング）」** する。
*   **リンク注文（OCO）のセット**:
    *   エントリー約定後、直ちに「利食い」と「損切り」の両方を予約し、監視不要にする。
*   **分割利食いとトレーリング**:
    *   最初のレジスタンスで半分利食いし「負けなし」を作る。残りは逆指値を切り上げて限界まで利益を伸ばす。
*   **パラボリックのトレイリングストップ（自動化）**:
    *   点（SAR）そのものを逆指値（ストップロス）ラインとして採用し、感情を排して利益を伸ばす。
    *   **タイムディケイ（時間管理）**: 価格が横ばいでも時間が経つとSARが接近するため、「ダラダラ保有」による機会損失を自動で防ぐ。

## 📝 議論・深掘り用スレッド
*ここに疑問やアイデアを書き足し、ブラッシュアップします。*

*   **[Q]**: 3日足の合成出来高を、兼業のルーティンにどう無理なく組み込むか？
*   **[Q]**: 逆指値のダブルチェック（11:30の解除）を自動化、または忘れず行う仕組みはどうする？
*   **[未昇華の課題: スイングアーキテクチャの副作用対策と覚悟]**:
    *   **資金効率の低下（機会損失）**: 待機状態の資金増加による資産曲線の停滞に対するポートフォリオ・資金分散戦略。
    *   **ドローダウンの長さ**: 勝率30〜40%、レンジでの連敗を乗り切るためのドローダウン許容額と資金管理（1%ルール）の徹底。
    *   **窓開け被弾（マクロショック）**: 週末などをまたぐ際のスリッページ（致命傷）リスクの許容と、1銘柄あたりの建玉上限による限定化。
    *   **含み益の蒸発とメンタル負荷**: 逆行中のチキン利食いをシステム的にどう防ぐか（建玉の分割決済やパラボリック追従への完全委譲など）。
*   **[未昇華のアイデア: 区間推定とノイズ排除のシステム実装]**:
    *   **タイミングと資金管理への極振り**: 難しいチャート分析（点推定）を捨て、タイミングと資金管理の2点だけで勝つ思想のプログラム化。
    *   **平均足への移行**: 細かなノイズ分析（ローソク足のヒゲなど）を視界から消し、方向感のみを示す「平均足」に変更する是非。
*   **[システムタスク: 短期/長期の自動スクリーニング実装アイデア]**:
    *   **週末スクリーニング**: 日々の監視を捨て、週末に条件合致を一括処理し、月曜朝のIFD注文（逆指値買い＋ロスカット）を準備する運用アーキテクチャ。
    *   **Python自動化要件**: 東証プライム等に限定し、`pandas`等のデータ分析環境を用いてBB(27)傾き・RCI(9, 81)のスクリプトを作成する。
*   **[システム要件: スクリーニングと執行ルールの厳密化]**:
    *   **ポジションサイジング公式**: `許容リスク額 ÷ (エントリー価格 - テクニカルな損切りライン) ＝ 適正株数` のシステム計算化。
    *   **スケーリング戦略**: 0か100かの全力取引をシステム的に禁止し、「打診買い（半分） → 増し玉 → 分割利食い」プロトコルを採用。
    *   **魔の時間帯回避**: 9:00〜9:15 / 12:30〜12:40 の逆指値狩り・ノイズを避けるための時限回避（11:30の一部解除・リザーブ注文等）ルール。
    *   **エネルギー充填の検知**: 波が発生する前の「BBスクイーズ」や「MAの収斂」を、エントリー必須条件の1つとしてフィルターに組み込む。
    *   **出来高の罠検知（Negative Prompt）**: 抜けがないのに急騰する値動きは「大衆釣りの罠（仕込み不足）」としてエントリー対象から除外する。
*   **[システムタスク: AI学習パイプライン（Log運用）の設計]**:
    *   **ジャーナリング要件**: Inboxに保留している個別の生ログ（M3, OLC等）から「気づき」を効率よく発掘し、マスタールールへ適宜昇華・デバッグするためのリフレクション（振り返り）ワークフローを別途確立する。
*   **[システム要件: エントリーの精度向上と絶対基準]**:
    *   **5日線付近の徹底**: 上昇トレンドの押し目において、エントリー位置は「必ず5日移動平均線の近く」であること。乖離している場合はいかなる理由でも見送る。
    *   **値幅の論理算定**: ボリンジャーバンドを用いたターゲット設定（基準線や+2σ）を利益計算の論理的根拠とし、欲による「感覚的な利確の先延ばし（含み益の蒸発）」を廃絶する。
    *   **日経平均との同期（マクロ順張り）**: 個別銘柄のセットアップが整っていても、日経平均の方向感（順張り）と一致しない場合は勝率が著しく落ちるためエントリーを見送る。

## 💻 【将来構想】自動スクリーナー／ML開発バックログ
*実装予定の技術タスクや機械学習のアイデア蓄積用*
*   **データ基盤の構築**: yfinance等のAPIを利用した日足データの自動取得と、`pandas`を用いた各種指標（BB, RCI, MACD）のバッチ計算・CSV出力パイプラインの構築。
*   **DDD（ドメイン駆動設計）導入**: トレードロジック（分析手法、画像処理・解析による入力値生成）をDDDアーキテクチャに乗せて実装するアーキテクチャ構想。
*   **予測MLモデル（オッズ計算）の開発**: 単純な「上がる・下がる」だけでなく、「複数のテクニカル指標の組み合わせから、目標値（利確ポイント）に到達する確率（オッズ）」を算出する機械学習モデルの実装。（「真のスイッチである確率」と「どこで止まるか」の定量化）
