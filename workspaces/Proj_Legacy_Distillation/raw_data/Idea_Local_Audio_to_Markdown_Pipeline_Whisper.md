# 概要
`Mail_Content_Converter` プロジェクトの動画・音声処理ワークフロー（旧構想）から切り出された、AIベースの音声文字起こし・要約パイプライン構想。

## 課題・背景
現状、グループコンサル等の長尺動画URL（m3u8等）から音声を自動抽出（`yt-dlp`）するスクリプトは完成し、その後の文字起こしはNotebookLMを用いた半手動運用（案A）でしのいでいる。
しかし今後のスケーラビリティを考慮すると、全てのアウトプットをローカル（またはAPIベース）で一貫して自動生成できる仕組みが必要。

## 次のアクション（アイデア）
Whisperを利用した全自動処理基盤の構築。

**技術スタック構想:**
*   `yt-dlp` (音声抽出)
*   `OpenAI Whisper` またはローカル推論環境（MacのMetalアクセラレーション活用）
*   `LLM API` (Claude 3.5 Sonnet / Gemini 1.5 Pro等による構造化)

**パイプライン設計案:**
1.  音声抽出スクリプト（既存）の出力をトリガーにする。
2.  音声ファイルをWhisperに渡し、プレーンなテキスト（またはタイムスタンプ付き）のフル文字起こしを取得。
3.  生のテキストデータをLLMに渡し、Zettelkastenに適した構造（概要、主要トピック、重要ポイントの見出し等）に自動再編させる。
4.  出力されたMarkdownを所定のディレクトリ（例：`30_Resources`）に格納。
