# キャリア戦略: AI Orchestrator 2026

## アクション・ロードマップ

### Phase 1: 基礎体力作り (現在 〜 1,2ヶ月)
* **目的:** AIを動かすための「共通言語」としてのPython操作に慣れる。
* **Action:**
    - [x] Python 3 エンジニア認定基礎試験 (取得済み)
    - [ ] [[Pythonデータ分析実務試験 学習ロードマップ（書籍・Udemy込み）|Python 3 エンジニア認定データ分析試験]] (Pandasによるデータ整形スキル確保)
    - [ ] 業務・学習でのAPI利用 (Requests, JSON操作)

### Phase 2: AIエンジニアリングへの拡張 (3ヶ月 〜 半年)
* **目的:** クラウド上でAIエージェントを動かす基盤を作る。
* **Action:**
    - [ ] **Python 3 エンジニア認定実践試験** (最優先：アプリ開発力の証明)
    - [ ] **AWS 認定ソリューションアーキテクト – アソシエイト (SAA)** (AIの置き場所・インフラ設計)
    - [ ] **[[G検定（AI基礎理論）学習ロードマップ|G検定]]** (AIの法律・倫理・全体像の把握 ※読み物として)
    - [ ] **フレームワーク学習:** LangChain, LlamaIndex, Streamlit

### Phase 3: 実績と権威付け (半年 〜 1年後)
* **目的:** 「作れる人」としてのポートフォリオと、上流設計の証明。
* **Action:**
    - [ ] **GitHubポートフォリオ:** 自作のAIツール（下記Idea参照）を公開し続ける。
    - [ ] **IPA システムアーキテクト (SA):** 要件定義・設計力の証明 (DBスペシャリストの次に狙う本命)。

## 資格戦略優先度

| 資格名 | 優先度 | 戦略的理由 |
| :--- | :--- | :--- |
| **Python 3 実践試験** | **最高** | 自動化ツールやボット開発の実務スキル（ログ、デバッグ、並列処理）に直結するため。 |
| **AWS SAA** | **最高** | AIをローカルからクラウドへ展開し、システムとして稼働させるために必須。DBの知識が活きる。 |
| **Python 3 データ分析** | 中 | データ整形の基礎として予定通り取得する。 |
| **G検定** | 中 | 読み物として活用。AI活用のリスク管理（著作権等）の引き出しを増やす。 |
| **Python 3 データ分析実践** | **低/保留** | モデル構築の深入りはAIに任せるため、優先度を下げる。 |
| **ITストラテジスト** | 低 | 経営視点よりも、まずは「実装・設計」の実績作りを優先。 |

## アイデア・インキュベーター (実装の種)

* **AntiGravity × Obsidian 自動連携**
    * Obsidianの特定フォルダを監視し、新規メモに対してAIが自動でタグ付け・関連リンク作成を行う。
* **自分専用 RAG (Knowledge Bot)**
    * 過去のツェッテルカステン、GitHubコード、Kindleハイライトを検索・回答できるチャットボット。
    * *技術スタック:* Python + LangChain + VectorDB (Chroma/Pinecone)
* **ドキュメント自動生成ボット**
    * GitHubのリポジトリを読み込み、コードの変更点を検知してドキュメント（READMEなど）を自動更新する。
