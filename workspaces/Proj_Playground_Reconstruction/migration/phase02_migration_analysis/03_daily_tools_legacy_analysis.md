# 03_daily_tools_legacy_analysis.md

## 目的
旧 `daily-tools` のスクリプトやツール群を棚卸しし、新アーキテクチャのどのレイヤーに配置すべきかを決定する。

## タスクリスト

### 1. ツールの棚卸しと利用頻度調査
- [ ] `daily-tools-legacy` 内のすべてのスクリプト・CLIツールをリストアップする。
- [ ] 現在も日常的に使用されているツールと、陳腐化または代替手段が存在するツールに分類する。

### 2. 新アーキテクチャへのマッピング
- [ ] 汎用的な機能やAPIラッパーとして有用なものは `core-service` （情報システム部・ライブラリ）への移行候補とする。
- [ ] Agentの自律実行の補助ツールとなるものは `agent-core` （Workspace用ツール）への移行候補とする。
- [ ] 人間（社長）がUIから直接操作する必要があるものは、`obsidian-view` または `mobile-vault` 用途として仕分ける。

### 3. 廃止・統合の判断
- [ ] To-Beアーキテクチャの導入により不要となるツールを特定し、廃止（Decommission）リストに追加する。

## 成果物
- daily-toolsの仕分け先マッピング表
- 廃止予定ツール一覧
