# 02_life_automation_legacy_analysis.md

## 目的
旧 `life-automation` リポジトリの密結合したスクリプト群を分析し、外部通信などの「副作用」と、タスク進行などの「実行ロジック」に分解・仕分けする。

## タスクリスト

### 1. 副作用（I/O）の特定と抽出
- [ ] 各自動化スクリプトが持つ外部API（Notion, Google Calendar, 外部サービス等）との通信箇所を特定する。
- [ ] DB操作やローカルファイルの直接読み書き（特にレガシーVaultへの直書き）を行っている箇所を特定する。
- [ ] 上記の副作用を `core-service` にカプセル化（Service化）するための関数・インターフェース設計案を策定する。

### 2. 実行ロジック・ワークフローの特定
- [ ] 定期実行（Cron等）で動作しているバッチ処理のトリガー条件をリストアップする。
- [ ] スクリプト内の条件分岐やタスク進行ロジックを特定し、`agent-core` の Agent ワークフローとして再構築できるか分析する。

### 3. 密結合ポイント（ハードコード）の洗い出し
- [ ] `second-brain-legacy` の特定のパス（例: `progress.md`）に依存しているハードコード箇所をリストアップする。
- [ ] 移行にあたり、DI（依存性の注入）や環境変数を用いて疎結合化するための要件を整理する。

## 成果物
- `core-service` 移行候補の副作用・APIリスト
- `agent-core` 移行候補のワークフロー・トリガーリスト
- リファクタリング対象のハードコードパス一覧
