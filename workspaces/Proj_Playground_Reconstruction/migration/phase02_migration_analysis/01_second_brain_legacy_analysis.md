# 01_second_brain_legacy_analysis.md

## 目的
レガシーな `second-brain` リポジトリを分析し、純粋な知識（Permanent Notes）と実行状態（タスク・ログ等のトランザクションデータ）を分離するための仕分けを行う。

## タスクリスト

### 1. 知識と状態の仕分け (Knowledge vs State)
- [ ] `10_Projects` や `20_Areas` 配下のディレクトリをスキャンし、稼働中のプロジェクトとアーカイブ済みプロジェクトを分類する。
- [ ] 各プロジェクト内の `progress.md` などの日々のタスク実行ログ（トランザクションデータ）を特定する。
- [ ] Zettelkastenとして `second-brain`（新）に残すべき普遍的な知識と、`agent-core/epics/` などに移行すべき「実行状態」のデータをリストアップする。

### 2. Inboxのノイズ分析と隔離
- [ ] `00_Inbox` 内のファイルを調査し、未整理のメモ（モバイルからの雑書き）と日々の運用データを分別する。
- [ ] 新しい `agent-core` で Issue/Epic としてキューイングすべきタスク系のメモを抽出する。

### 3. スクリプト・実行ロジックの混入チェック
- [ ] `second-brain-legacy` の奥深いディレクトリに置かれたままになっている自動化スクリプトやバッチファイルがないかスキャンする。
- [ ] 発見した場合、`core-service`（副作用のカプセル化）または `agent-core`（実行層）への移行候補としてリストアップする。

## 成果物
- 知識抽出対象リスト (Markdown/CSV)
- `agent-core` へのタスク・ログ移行リスト
