# Data Cleansing Plan (移行前の必須タスク)

新アーキテクチャへ移行（箱の移動）を行う前に、既存データの「負債」を清算し、Agentが処理可能な状態へリセットする。

1. **進行中 Project の棚卸しと切り戻し**
   - 現在稼働中の `10_Projects` 内のプロジェクトを全て見直し、一旦「Paused（中止）」状態とし、未完了の作業は `agent-core` 側で管理すべき「タスク」または「Backlog」として抽出・切り戻す。
   - プロジェクト内の `progress.md` などの実行ログはアーカイブするか破棄し、プロジェクトフォルダ内には「決定事項（正本）」のみを残す。
   - 完了済みプロジェクトは "Distill or Delete" フローに従い、教訓を抽出し `Zettelkasten` / `Areas` に蒸留した後、元のファイル群を完全削除する。
2. **`00_Inbox` のクレンジング**
   - 現在のInboxに散乱している一時メモや自動生成レポート（Harvest Report等）を精査し、必要なものはZettelkasten化（Resources）またはプロジェクトへ移動。不要なものは破棄し、Inboxを空（ゼロ）にする。
3. **`20_Areas` の棚卸しとフォーマット適用**
   - 旧体系における `20_Areas`（またはそれに類するガバナンス文書）を見直し、新アーキテクチャの必須YAMLメタデータ（ラショナールなど）のフォーマットに合わせて一括リライトし、新 `10_Areas` へ移行する。
4. **`30_Resources` と `50_Zettelkasten` の分離**
   - 既存の `30_Resources` 内にある「Permanentノート（すでに抽出・独立した知識）」をピックアップし、フラット化して新 `Zettelkasten` へ移動する。
   - 生の外部資料などは新 `Resources` に移行する。
4. **`40_Archives` の解体**
   - アーカイブフォルダを順次見直し、価値ある知見（アンチパターンなど）を蒸留し、残りは全て破棄（rm -rf）する。
5. **システム領域の移行**
   - `Templates` と `Attachments` は新 `Meta` へ移動する。
   - `Automations`（実行スクリプト）等の周辺ファイルは実行層へ退避させる。
