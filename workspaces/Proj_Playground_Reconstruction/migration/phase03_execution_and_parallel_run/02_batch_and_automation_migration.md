# 02_batch_and_automation_migration.md

## 目的
レガシーの自動化・バッチ処理を `agent-core` 上のAgentワークフローとして再構築し、並行稼働検証を実施する。

## タスクリスト

### 1. `agent-core` ワークフローの構築
- [ ] 必要なAgentのペルソナ定義と、実行する Workflow のスクリプト/ルールを定義する。
- [ ] 旧バッチ処理のロジックを、Agentがタスクキュー（Issue/Epic）を起点に処理するフローとして再実装する。

### 2. `core-service` の統合とサンドボックス検証
- [ ] 再実装したワークフロー内で、`core-service` モジュールを呼び出す（DI）ように連携を実装する。
- [ ] `workspaces/` ディレクトリ配下に展開される Docker サンドボックス上での実行テストを行う。

### 3. Legacyからの切り替え
- [ ] 現行で稼働している機能（並行稼働を要するバッチ）は存在しないため、並行稼働テスト（Dry-Run等）は実施しない。
- [ ] 新環境での実装とテストが完了し次第、移行完了まで居座らせていた `-legacy` リポジトリからの切り替え（廃止）を宣言する。

## 成果物
- デプロイ済みの `agent-core` ワークフロー
