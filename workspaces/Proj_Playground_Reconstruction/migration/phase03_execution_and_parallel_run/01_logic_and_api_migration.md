# 01_logic_and_api_migration.md

## 目的
Phase 02 の分析結果に基づき、副作用を持つ処理を `core-service` へ移植・実装する。

## タスクリスト

### 1. `core-service` の基盤構築
- [ ] `core-service` リポジトリの基本ディレクトリ構成を作成する。
- [ ] 品質ゲートとしての CI/CD パイプライン（Lint, ユニットテスト実行基盤）をセットアップする。

### 2. 副作用（API通信・DB操作）のService化実装
- [ ] レガシーリポジトリから抽出したAPI通信・ファイル操作処理を、ステートレスな Service クラス/関数として実装する。
- [ ] 各 Service に対して、必ず仕様化テスト（ユニットテスト）を記述し、副作用の動作を保護する。
- [ ] ハードコードされていた認証情報等を環境変数やシークレットマネージャ管理に移行する。

### 3. DIインターフェースの整備
- [ ] `agent-core` 展開時に動的DI（依存性の注入）が可能な形でモジュールをエクスポートし、疎結合性を担保する。

## 成果物
- 実装・テスト済みの `core-service` モジュール群
- Serviceインターフェース定義書
