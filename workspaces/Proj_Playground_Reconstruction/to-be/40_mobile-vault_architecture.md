# 40_mobile-vault アーキテクチャ設計書

## 1. 概要と責務
`mobile-vault` は、「You, Inc.」における社長（人間）のための最も軽量なインターフェース（Thin Client）です。
iCloudなどを介してスマートフォンやタブレットと同期されることを前提とし、システムやAgentのための複雑な設定ファイル（READMEやReferenceなど）は一切持ちません。

純粋に「社長が雑記を投下する場所」「個人的なメモを置く場所」「Agentの報告を見る場所」の3つのみで構成されます。

## 2. ディレクトリ構成とアクセス境界

Agentの管轄領域と、不可侵領域（プライベート）をディレクトリ単位で厳格に分離します。

```text
mobile-vault/
├── 00_Inbox/               # 📥 【Agent回収対象】社長が雑記・タスク・思いつきを投げ込む場所
├── 10_Private_Notes/       # 🔒 【Agent立入禁止】社長個人のメモ置き場。回収・読み書き不可の絶対不可侵領域
└── 20_Dashboards/          # 📊 【Agent出力専用】Agentが Today.md や要約レポートをパブリッシュする場所
```

## 3. コア機能とルール

### 3.1 00_Inbox（人間からAgentへの一方通行入力）
- 人間はスマートフォンから、思いついたアイデアやタスクをここに乱雑に作成します。
- `agent-core` 側のバックグラウンドジョブ（InboxTriageAgentなど）が定期的にここを監視し、ファイルを回収（Move）して `second-brain/00_Inbox/` に清書して転送します。
- 回収されたファイルは `mobile-vault/00_Inbox/` から自動的に消去され、常にクリーンな状態が保たれます。

### 3.2 10_Private_Notes（絶対不可侵の聖域）
- Agentに見せたくない個人的なメモや、回収されずにずっと手元に置いておきたい情報はすべてここに保存します。
- `agent-core` 側の実装において、このディレクトリへの Read/Write アクセス権限はシステムレベルでハードブロックされます。

### 3.3 20_Dashboards（Agentから人間への一方通行出力）
- Agentが `second-brain` 内のタスクやスケジュールを集約し、モバイルで読みやすいフォーマット（例: `Today.md`, `Weekly_Report.md`）にコンパイルして出力する先です。
- ここにあるファイルはすべてAgentによって自動生成・上書きされる「Read-Only View」であり、人間が手動で編集しても次回のビルドで上書きされます。
