# Agent OS Inbox & Portfolio Architecture

このドキュメントは、Life Automation プロジェクト（Phase 6以降）における「人間とAgentの協働エコシステム」の根幹である、双方向のInboxパイプラインアーキテクチャと、Life Portfolioを用いたPDCAサイクルの運用ルールを定義するものです。

## 1. 思想 (Philosophy)
Inboxは単なる「未処理メモのゴミ箱」ではありません。
それは、人間とAgent（AI）が同じファイルシステム（second-brain）を介して、非同期でお互いにタスクを依頼し合い、アイデアを熟成させるための**双方向のパイプライン（流れる川）**です。
データの流れを視覚的に強制するため、ディレクトリには `00_` から順にプレフィックスが付与されています。

## 2. ディレクトリ構成と窓口の責務 (The Inbox Pipeline)

### 📥 00_Drop_Zone (カオスの受付)
- **場所**: `second-brain/00_Inbox/00_Drop_Zone/`
- **役割**: モバイルからのメモやPCからの思いつきなど、未処理・未整頓のデータが着弾する最初の受け皿。
- **アクション**: 毎朝 PM Agent がここを走査し、中身を意味解釈して以下の窓口へディスパッチ（仕分け）する。処理後の元ファイルは削除され、Inbox Zeroに保たれる。

### 🤖 01_Agent_Queue (AIへの依頼窓口)
- **場所**: `second-brain/00_Inbox/01_Agent_Queue/`
- **役割**: 「競合の最新プレスリリースを要約して」「このコードをリファクタリングして」など、Agentが自律実行できる非同期タスクの待ち行列。
- **アクション**: Worker Agent（リサーチャーやコーダー）がバックグラウンドでここを監視し、タスクがあれば勝手に実行する。

### 🧑‍⚖️ 02_Review_Queue (人間への報告窓口)
- **場所**: `second-brain/00_Inbox/02_Review_Queue/`
- **役割**: AIが実行したタスクの完了レポートや、システムが自律生成した Harvest Report など、「人間のレビューと承認」を待つ検疫エリア。
- **アクション**: ユーザーが内容を確認し、問題なければアーカイブ、またはシステムの教訓として採用（永続化）する。

### 💡 03_Idea_Backlog (アイデアの育成窓口)
- **場所**: `second-brain/00_Inbox/03_Idea_Backlog/`
- **役割**: PM Agentのトリアージによって「今はタスクではないが価値のあるアイデア・ナレッジ」と判定されたメモの退避場所。
- **アクション**: 週末などに CIO Agent（または人間）がここを見返し、体系化されたナレッジとして `20_Areas/` 配下の永久ノートへと昇華させる。

### 🏃‍♂️ Inbox_Tasks.md (人間のタスクプール)
- **場所**: `second-brain/00_Inbox/Inbox_Tasks.md`
- **役割**: 「市役所に行く」「顧客とミーティング」など、物理世界や人間にしか実行できないタスクのリスト。
- **アクション**: COOモジュール（Daily Planner）がここからタスクを拾い、カレンダーや今日の計画に配置する。

## 3. Life Portfolio カテゴリ（人生の投資先）
抽出されたタスク（人間・AI問わず）には、人生のどの領域へ時間を投資しているかを可視化するため、以下の4つの **Life Portfolio** カテゴリが付与されます。

1. 💼 **`(Work)`**: 顧客への価値提供、本業、直接的な収入に結びつくタスク。
2. 🌱 **`(Growth)`**: 学習、自己研鑽、中長期的な自身の価値を高める未来への投資。
3. 🔧 **`(Maintenance)`**: 家事、手続き、健康維持など、生活の基盤を維持・防衛するタスク。
4. 🎮 **`(Play)`**: 趣味、エンタメ、心身のエネルギーを回復・チャージする休息タスク。

---

## 4. Life Portfolio による予実管理と PDCA (Review Workflow)
Agent OS は単なるタスク実行に留まらず、Life Portfolio を用いて「意図的に余白や休息を確保する」ためのフィードバックループ（PDCA）を自律的に回します。

1. **Plan (計画)**: COOモジュールが、タスクのカテゴリと見積もり時間（`est`）を考慮し、Focus（上位目標）にアラインした1日のスケジュールを組む。
2. **Do (実行)**: 人間と Worker Agent が各タスクを消化する。
3. **Check (分析)**: 週末の Weekly Review にて、CFO / PM Agent が完了したタスクの実績を集計し、「今週は Work に80%投資し、Play が0%だった」などのポートフォリオ分析レポートを生成する。
4. **Action (改善)**: 分析結果をもとに、Agentが「来週の Focus には Play（休息）を強制的に20%組み込みましょう」と提案し、次の Plan の質を向上・調整する。
