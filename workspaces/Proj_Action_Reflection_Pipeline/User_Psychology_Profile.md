# User Psychology Profile & Core Needs

本ドキュメントは、Epic 03「Action & Reflection Pipeline」を構築する上での**最も重要なシステム要件（ユーザーの深層心理と満たすべきニーズ）**を定義したものです。今後のAgent設計やプロンプト設計において、全てのベースとなる「北極星（コンパス）」として機能します。

## 1. ユーザーの現状と特性（The Baseline）
* **極端さとストイックさ**: 「成長（未来のため）」への時間投下は楽しく充実感を得られる反面、ストイックになりすぎて「遊ぶ（休む）」ことを後回しにしてしまう特性がある。
* **迷子状態（現在）**: かつては「早起きして勉強してから出社する」という日々に**充実感や「無敵感」**を感じていたが、現在はモチベーションが低下し、昼まで寝てしまうなどフォーカスが定まっていないことに焦りと恐れを感じている。

## 2. システムが解決すべき「真の課題」
ユーザーは「遊びたくない」わけではない。最大のブロックは、**「資格勉強などの『やるべきこと（Should）』の進捗が順調であるという『安心』が見える化されていないため、心から『やりたいこと（Want）』を楽しめない」**という構造にある。
スケジュールを感覚だけで組んでいるため、常に「まだ足りないのではないか」という不安があり、結果として勉強に偏りすぎて摩耗してしまう。

## 3. システム・Agentに求められる振る舞い（Requirements）

### A. Should と Want の明確化と分離
* タスクを単一のリストとして扱うのではなく、**「Should（資格勉強などの未来への投資）」**と**「Want（ギター、旅行などの純粋な喜び・余暇）」**を明確に区別して管理する。

### B. 「安心（Safety）」の見える化による免罪符発行
* システムは、Shouldタスクの進捗状況をトラッキングし、ユーザーに明示的に提示する。
* 「これだけShouldを進めているのだから、今はWantをして休んでいい」という**論理的な安心感（免罪符）**を与えることで、ユーザーに良質な余暇（トランジション）を取らせる。

### C. 「無敵感」ルーティンの自動修復
* かつてユーザーに大きな充実感を与えていた「朝の黄金ルーティン（早起き・勉強）」を、Agentが少しずつカレンダーに組み込み、ユーザーのモチベーションと無敵感を取り戻すためのペースメーカーとなる。

## 4. 目指すべきゴール
**「ShouldとWantを適切にブレンドしてゴールへ導きつつ、進捗の安心感を担保することで、ユーザーの精神的摩耗を防ぐ自律型パートナーシステム」**の構築。
