# Inbox Operations Policy

## 1. 目的 (Purpose)
本ポリシーは、`01_Inbox`（一時保管庫）を常に高効率かつクリーンに維持し、情報洪水を防ぐと同時に、ユーザーを「整理の負債」から解放するためのインボックス運用ルールを定義します。

---

## 2. ディレクトリ構造とトピックの再定義

`01_Inbox` 配下は、部門ごとの複雑な分類を完全に廃止し、以下のシンプルな役割別フォルダで構成します。**すべての移動および初期受け入れは、Inbox配下のフラットな構造または専用レシーバー内で完結させます。**

```text
01_Inbox/
├── (直下)               # 分類不明メモ、新規一時メモの保留場所
│                        # ※ここにあるメモは、定期的に「マージ・統合」または「Zettelkasten化」を検討する
│
├── fleeting/            # 日々のFleeting notes、一時的な落書き・雑記（旧00_Dailyの代替）
│                        # ※日誌や気付きはすべてここに集約し、直下を汚さない
│
├── Trade/               # トレードログ・相場分析専用のレシーバー
│                        # ※自動/手動生成されたトレード関係はここに集約する
│                        # ※PC側の「mobile_sync」スクリプトが自動的にここに振り分けを行う
│
├── Backlog/             # 将来のアイデア・未実行タスクの待合室
│                        # ※「Idea_...」や「Backlog.md」など、将来プロジェクト化したい拡張案の保管庫
│
└── Defrag/              # 自動デフラグ機能（Zettelkastenデフラグなど）のアウトプットレシーバー
```

> [!NOTE]
> **旧 `00_Daily/` の完全廃止**
> 思考の作業場であった `00_Daily` ディレクトリは廃止され、その機能は `01_Inbox/fleeting/` に完全に統合されました。これにより、すべての「未完成・一時的なメモ」は `01_Inbox` の処理フローに統一されます。

---

## 3. 自動振り分けの仕組み (mobile_sync 連携)

モバイルデバイス（iCloud Obsidian）から同期されるファイルは、`life-automation` リポジトリの `move_obsidian_notes.sh` を通じて PC 側に同期されます。
*   **Trade日誌の自動集約**: 同期されるファイル名に `Trade` が含まれる場合、自動的に `01_Inbox/Trade/` フォルダへ格納されます。
*   **その他のMarkdownファイル**: `01_Inbox/` 直下（または指定されたプレフィックスに応じて fleeting 等）へ振り分けられます。

---

## 4. クリーンアップおよびマージ運用フロー

インボックスの肥大化を防ぐため、以下のフローに従って定期的に大掃除（Inbox Zero）を行います。

| 収集したノートの性質 | 一時保管先 | 処理・最終的な帰属先 (PARA/Zettelkasten) |
| :--- | :--- | :--- |
| **アクション・プロジェクト** (期限・ゴールあり) | `01_Inbox/Backlog/` | 準備ができ次第、`10_Projects/` にプロジェクトとして昇格 |
| **一時的な落書き・日常ログ** (Fleeting) | `01_Inbox/fleeting/` | 内容を精査し、Zettelkasten化するか不要になれば削除 |
| **事実としてのログ（Trade等）** | `01_Inbox/Trade/` | 週次で分析し、教訓を `30_Resources/` へ還元後、ログは `40_Archives` へ |
| **分類不明・ドラフトメモ** | `01_Inbox/` (直下) | **[マージ検討]** 類似したテーマのメモは積極的に統合し、最小限の Claim ベースノートに昇華させる |

### 🔍 分類不明メモのマージポリシー
直下に残る複数の「分類不明メモ」は、放置せず以下の基準でマージ・整理します：
1.  **類似テーマの統合**: 似たようなトピックのメモは1枚のノートにマージする（例：「教える力育成の第2回」「教える力育成の第3回」 -> 「教える力育成_まとめ」）。
2.  **Zettelkastenへの昇華**: 統合・清書されたメモから「普遍的な主張（Claim）」を抽出し、`30_Resources/Permanent/` へ Permanent Note として昇格させる。
3.  **不要なドラフトの破棄**: 原則が抽出された後の元のログやゴミファイルは、アーカイブせず積極的に削除する。
