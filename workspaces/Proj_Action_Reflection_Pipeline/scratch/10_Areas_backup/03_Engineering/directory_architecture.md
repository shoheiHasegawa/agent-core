# Directory Architecture Policy

## 1. システム・アーキテクチャとディレクトリ構成
「行動（PARA）」と「思考（Zettelkasten）」を両立させるハイブリッド構成。

```text
second-brain/
├── 01_Inbox/          # 【一時保管庫】未処理のメモ、Webクリップ、切り出したアイデア
├── 10_Projects/       # 【行動：期限あり】現在進行中のプロジェクト（PARA）
├── 20_Areas/          # 【責任：期限なし】自分株式会社（You, Inc.）のC-Level管轄
├── 30_Resources/      # 【知識：興味関心】第2の脳のコア（Zettelkasten格納場所）
├── 40_Archives/       # 【過去】完了したプロジェクト、凍結した資料
└── 99_System/         # 【裏方】テンプレート、画像（Attachments）、システム設定
```

### Area (20_Areas) と Project (10_Projects) の責務分離ルール
- **Project (`10_Projects`)**: 完了条件と期限があるアクションプラン。完了後は `40_Archives` へ移動する。複数の部門を横断する機能があるため、各部門のディレクトリ下には置かず独立させる。
- **Area (`20_Areas`)**: ライフサイクル（終点）がなく、高い基準で維持し続けるべき責任領域。この中には進行タスクを置かず、「SOP（手順書）」「長期ロードマップ」「意思決定ログ（Decision Log）」「設定ファイル」などの**資産・マニュアルのみ**を置く。

### You, Inc. 各部門 (C-Level) の管轄
- **CEO**: 戦略的に重要な特命プロジェクト、経営理念(MVV)、複数部門にまたがる人生の意思決定（キャリア、居住地など）。
- **CFO**: 財務（投資・家計）、キャッシュフローの管理（3つのバケツ）、投資判断や金融機関連携。
- **CMO**: 広報・マーケティング、UI/UX（外見・スキンケア・ファッション）、物理的な外部防御とメンテナンス、人間関係の戦略。
- **CHRO**: 人事・福利厚生、健康（睡眠・SPOF保護）、体内環境の最適化（食事・栄養・腸活）、休息ルールの決定。
- **CIO**: Zettelkasten運用、ツール設定、AI連携やシステム・リポジトリ構成の技術的決定。
- **GA**: 総務（General Affairs）、分類不能な雑務用バッファ（迷ったらここかInboxに入れる）。

### SOP（手順書）のルール
- **SOP (Standard Operating Procedure)**: 「今、どう動くべきか」の最新手順。ファイル名に日付は入れず、常に上書き更新する。
- 過去の意思決定の経緯は `40_Archives/Decisions/` のADR（Decision Log）へ参照を促す。
