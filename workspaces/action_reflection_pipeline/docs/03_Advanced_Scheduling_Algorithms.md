# 03. Advanced Scheduling Algorithms

単なる「タスクの消化（TODOリスト）」ではなく、CEOが最高のパフォーマンスを発揮し、かつ確実にゴールへ到達するための**「秘書エージェントの思考回路（スケジューリングアルゴリズム）」**となる絶対ルール（制約）の定義です。
これらの制約は、ドメイン層のロジックとしてコードに実装されます。

## 1. 認知負荷の最小化と実行効率（Cognitive Load & Flow）
**「CEOの脳の切り替えコストを削り、決断疲れをなくす」**ための制約群。

*   **1-A: WIP（仕掛かり）の厳格な制限**: 
    1日の「主要タスク（MUST）」は最大 **3つ（WIP_LIMIT=3）** までとし、同時並行（マルチタスク）を禁止。確実に「終わらせてから次を始める」フローを強制します。
*   **1-B: コンテキスト・バッチング（切り替えコストの排除）**:
    深い思考が必要な作業（DeepWork）と、浅い作業（ShallowWork / メール等）をスケジュール内で自動的にブロック化し、1日の中での「深い⇔浅い」の往復（コンテキストスイッチ）を**最大3回まで**に抑えます。
*   **1-C: 未Readyタスクの完全な不可視化**:
    「Aが終わらないとBができない」依存関係がある場合、Aが完了するまでBの存在を今日のスケジュールや視界から完全に消去し、無意識の認知リソース消費を防ぎます。

## 2. 逆算思考と目的ドリブン（Backcasting & PM）
**「積み上げではなくゴールから逆算し、戦略的な前進を担保する」**ための制約群。

*   **2-A: クリティカルパス逆算制約 (Latest Finish Date)**:
    タスクに `deadline`（期限）と依存関係を持たせ、それらから「遅くとも終わらせるべき日（LFD）」と「実行予定日（target_date: バッファを含めた逆算日）」を自動算出し、間に合わない場合は警告します。
*   **2-B: 戦略的投資枠（第2象限）の強制ブロック**:
    1日の空き時間の **20%** は必ず「重要だが緊急ではないタスク（Should: 自己研鑽や中長期戦略）」のために強制的に先取りブロックし、緊急タスクによるリソースの枯渇を防ぎます。
*   **2-C: 孤立タスク（Orphan Task）の排除**:
    上位の目的（Areas）に紐付いていない思いつきのタスクは、Inboxに留め置き、目的が明確化されるまでは本スケジュールに組み込ませません。

## 3. 計画的余暇とウェルビーイング（Well-being & Leisure）
**「人間のエネルギーを有限とみなし、持続可能なハイパフォーマンスを維持する」**ための制約群。

*   **3-A: リカバリー・ファースト制約**:
    タスクを埋める前に、まずは必ず「十分な睡眠ブロック」と「最低1時間の [W] Want / 余暇ブロック」を固定配置します。休む時間を「残り時間」からひねり出す設計を禁止します。
*   **3-B: 超日リズム（Ultradian Rhythm）に基づく連続稼働リミット**:
    **90分以上**のディープワークの連続を禁止し、90分後には必ず15〜20分の「強制ブレイク」または「低負荷・[W] タスク」を挿入させます。
*   **3-C: [M] と [W] のバランス（動機づけの維持）**:
    スケジュールにおける `[W] Want` タスクの割合が **20%** を下回った場合、「義務（Must）に押しつぶされる危険性」として警告フラグを立て、CEOにリフレッシュを促します。

## 4. 時間的特性とエネルギーマッピング（Chronobiology & Energy Mapping）
**「タスクの性質と、人間の生体リズム（脳のエネルギー残量）を完全に一致させる」**ための制約群。

*   **4-A: 朝のディープワーク強制（Eat That Frog 制約）**:
    脳の前頭葉が最もフレッシュな「起床後〜昼食まで」の午前中のゴールデンタイムには、タグ付けされた `[Energy: High]`（深い思考、戦略立案、重いコーディングなど）のタスクのみを優先配置し、この時間に浅い作業（Shallow Work）を入れることを禁止します。
*   **4-B: 午後のサーカディアン・ディップ（Circadian Dip）回避**:
    昼食後の13:00〜15:00の認知能力が低下する時間帯には、あえて `[Energy: Low]`（ルーチン作業、連絡・承認作業）や物理的タスク、もしくは割り切って `[W]` の余暇ブロックを自動配置します。
*   **4-C: シャットダウン・リチュアル（1日の終結儀式）の固定**:
    就業時間（または1日の活動終了）の最後の30分間は新規タスクを入れず、「明日の計画（ジャーナリング）」と「脳のクールダウン」のための固定ブロックとし、仕事の文脈を睡眠に持ち込ませません。
