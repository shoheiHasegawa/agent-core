# 🤝 ステークホルダー・エンゲージメント (Stakeholder Engagement)

勉強会取りまとめの担当者へのヒアリングおよび導入合意形成のためのアクションプランです。

---

## 🎯 ヒアリング方針（2026/04/20 更新）
**「運営コストの大幅削減（手作業の撲滅）」をメインテーマとする**

*   **主目的**: 面倒な「目視確認」「手動色塗り」「集計転記」をゼロにし、運営側の負担を最小化すること。
*   **サブ目的（発展形）**: 運用が回り出した結果として、評価データ（リアル参加率等）も自動的に集まるようになること。

---

## 📩 アポ依頼メッセージ案

**件名: 勉強会運営の「手作業コスト」削減に関するご相談（出欠管理の自動化案）**

お疲れ様です。〇〇です。

現在、勉強会の運営に関わる中で「Excelでの出欠管理（目視確認や手動の色塗り、集計転記）」が、運営側の小さくない負担になっていると感じています。

これらの「面倒な手作業」をGoogleフォームとスプレッドシートを連携させて完全に自動化し、運営コストをほぼゼロにするための仕組みを検討しています。

つきましては、現在のExcel運用で「特にどの作業が手間に感じているか」「新システムに切り替える上での懸念」など、15〜20分ほどヒアリングさせていただけないでしょうか。

この仕組みが導入できれば、運営側の当日の拘束時間を減らせるだけでなく、集計ミスもゼロにできると考えています。
（将来的な話として、評価データの抽出なども効率化できる副次的なメリットもあります）

ご多忙のところ恐縮ですが、ご検討いただけますと幸いです。

## 📋 ヒアリングアジェンダ

1.  **現行Excel運用の「負」の深掘り**:
    *   当日、一番「手間だ、面倒だ」と感じている瞬間はどこか？
    *   色塗りや記号入力に、独自の暗黙ルール（隠れた仕様）はないか？
    *   数式の破損や、集計漏れでヒヤッとしたことはないか？
2.  **募集・登録プロセスの検討（New）**:
    *   受講生募集フォームで「何を必須項目にするか」のすり合わせ（名前、社員番号、拠点[所属]、会社メアド[将来の拡張用]など）。
    *   拠点情報の管理レベルの確認（部署まで細かく必要か、拠点単位で良いか）。
3.  **新システムへの移行ブロッカー特定**:
    *   「システム化することで、逆にマスタの管理が大変にならないか？」という不安はあるか？
    *   イレギュラーな事態（当日キャンセル、飛び入り参加）にどう対応しているか？
4.  **既存表の「役割」の棚卸し**:
    *   既存のExcelを「運営以外」の誰が見ているか？（その人たちに迷惑がかからないか？）
5.  **期待値調整・デモ**:
    *   「まずは運営を楽にする」ためのプロトタイプ紹介。

## 📄 説明資料の骨子案

1.  **現在の課題**: 「運営を圧迫するExcelの目視・手入力運用」。
2.  **解決策**: 「Google Formsによる打刻 ➔ スプレッドシート自動判定」。
3.  **Before/After**:
    *   Before: 毎回の出欠確認10分 ＋ 手動色塗り ＋ 月次集計。
    *   After: **受講者が打刻 ➔ 運営は見るだけ（0分）**。
4.  **発展的なメリット**: 運用が安定すれば、リアル参加率の集計なども勝手に行われるようになります。
5.  **想定されるリスクと「保険（対策）」**:
    *   管理者が操作ミスをしても、**「ボタン一つで修復できるリカバリ機能」**を準備していること。
    *   システムがブラックボックス化しないよう、**「設計図（コード・仕様）」をAI等で誰でも読み解ける形**で管理していること。
    *   （誠実さの提示：デメリットも理解した上で、対策を打っていることを伝えて信頼を得る）
